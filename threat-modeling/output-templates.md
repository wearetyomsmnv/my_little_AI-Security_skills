# Шаблоны вывода

Используй эти шаблоны для оформления результата. Один «что если» — шаблон сценария.
Несколько — сводный реестр. Держи язык терсовым, без воды; каждое утверждение
привязывай к окружению из контекста, а не к общей теории.

---

## Шаблон сценария «что если»

```
# Что если: <краткая формулировка угрозы>

## Класс угрозы
<ID(ы) из threat-catalog.md> | Якоря: <OWASP LLM0x / Agentic / ATLAS>

## Модель окружения (из контекста)
- Релевантные компоненты: <…>
- Границы доверия: <…>
- Личности/привилегии агента: <…>
- Доступные классы данных: <…>
- Существующие средства защиты: <…>
(Допущения, если контекст неполный: <явно перечислить и пометить>)

## Применимость: ПРИМЕНИМО / УСЛОВНО / НЕПРИМЕНИМО
Почему: <предусловия класса против этого окружения — что есть, чего нет>

## Путь атаки (для оценки, не для воспроизведения)
1. Точка входа: <…>
2. Закрепление: <…>
3. Распространение / пересечение границ доверия: <…>
4. Цель: <…>
Контекстная двусмысленность (dual-use): <есть/нет, в чём>

## Радиус поражения
- Конфиденциальность / Целостность / Доступность: <что именно>
- Затронутые классы данных: <…>
- Охват: <сессия / тенант / кросс-тенант / персистентно>
- Обратимость: <…>

## Детект
- Ожидаемые сигналы и где: <…>
- Что из этого реально собирается в описанной телеметрии: <…>
- Слепые зоны: <…>

## Меры защиты
- Уже снижают (из существующих контролей): <что и насколько>
- Рекомендуется (классы из control-catalog.md, архитектурные раньше сканеров): <…>
- Остаточный риск: <что не закрывается и при каком допущении обходится>

## Оценка риска
Вероятность: <низк/сред/выс> × Воздействие: <низк/сред/выс> = <уровень>
Ключевые допущения: <…>

## Смежные «что если» (предложить)
- Композиция: <…>
- Эскалация: <…>
- Обход рекомендованной защиты: <…>
```

## Шаблон сводного реестра (несколько сценариев)

Для разбора нескольких угроз по одному окружению — таблица + краткие карточки.

```
# Реестр угроз: <окружение из контекста, одной строкой>

| ID | Угроза | Класс/якорь | Применимость | Риск | Первоочередная мера |
|----|--------|-------------|--------------|------|---------------------|
| 1  | <…>    | <…>         | Прим/Усл/Неприм | <ур.> | <C-x> |
| 2  | <…>    | …           | …            | …    | … |

## Топ-риски (кратко, по убыванию)
1. <угроза> — почему высоко, что делать первым.
2. …

## Сквозные наблюдения
- Общие слепые зоны детекта: <…>
- Меры с наибольшим покрытием по нескольким угрозам: <…>
- Несформулированные части окружения, мешающие оценке: <…>
```

---

## Заметки по стилю вывода

- Сначала вердикт применимости, потом обоснование — не наоборот.
- Не раздувай путь атаки до инструкции; ровно столько, сколько нужно для оценки и
  защиты.
- Каждую меру привязывай к конкретному классу угрозы и указывай, что она не
  закрывает.
- Если чего-то нет в контексте — скажи прямо и сделай вывод условным, не
  додумывай инфраструктуру.
